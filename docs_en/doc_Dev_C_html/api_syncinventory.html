Synchronized inventory

To implement synchronous inventory using the TagInventory_Raw and GetNextTag functions, the TagInventory_Raw function requires a specified time parameter to inventory tags. The tags read are buffered inside the reader. Once called, this function will block until the timeout period ends, then return the number of tags read. In principle, no other RFID operation interfaces should be called during this blocking period.The GetNextTagfunction is then used to retrieve the tags from the buffer. The number of times GetNextTag must be called should match the number of tags returned by TagInventory_Raw.

Synchronous inventory is a suitable choice when short-duration inventory operations are needed, and the number of tags is not very large.

Since the reader's internal buffer has a limited capacity for tags and TagInventory_Raw is a blocking function, inventorying a larger number of tags in Normal Mode generally requires using a timer or a thread to call the function in a loop. Appropriate idle time should be incorporated into the loop to consider heat dissipation and hardware wear.

Note:​​ For previous-generation readers like the M6e type, the MTR_PARAM_TAG_INVPOTL parameter must be set before calling the aforementioned functions; otherwise, an error will be returned. Current reader models have a default value for this parameter, making it a non-mandatory configuration item.
