Reader status

After power-on, the reader enters the ​​bootloader state​​. At this stage, the API and the reader are in a ​​non-connected state​​, and only firmware upgrade operations can be performed. The firmware upgrade can be executed using the interface function FirmwareLoadFromSerialPort. RFID-related operations are not available in this state. Once the interface InitReader_Notype is called to initialize the reader, the reader transitions to the ​​application layer state​​, also known as the ​​ready state​​. In this state, the API and the reader establish a connection, and all related RFID operations can be performed. If the link between the API and the reader is disconnected during the ready state, subsequent calls to interface functions will return relevant errors.
If an exception occurs during the execution of an RFID interface function, it will return an appropriate error code immediately. This error code can be analyzed to determine the current connection status between the API and the reader. Minor errors typically do not terminate the connection, whereas severe errors will disrupt the link, requiring re-initialization of the reader.When an asynchronous fast mode inventory is invoked in the ready state, the reader enters the ​​operational state​​. In this state, all exceptions are reported via callbacks. If an exception occurs, the reader stops operations and returns to the ready state. However, in cases of hardware issues such as power instability causing a reader reset, the reader might revert directly to the ​​bootloader state​​, breaking the connection.The connection status between the API and the reader can be directly checked by calling parameter acquisition interface functions. A successful return indicates the connection is online; otherwise, the connection is broken.
