ModuleAPI_C Introduction

ModuleAPI C provides a development interface for the C language, applicable to operating system platforms such as Windows, Linux, and Android.

ModuleAPI C is interfaced as a dynamic library for developers to reference. Developers can use the interface functions in the dynamic library to drive the reader. The names of the ModuleAPI C dynamic libraries vary slightly across different platforms. For the Linux operating system platform, the dynamic library is named libModuleAPI.so, providing 32-bit or 64-bit versions compiled for CPUs based on the x86-64 instruction architecture. For Linux systems based on other instruction architectures, technical support should be contacted. The Windows operating system platform provides 32-bit or 64-bit dynamic libraries based on x86-64. The 32-bit dynamic library is named ModuleAPI_C.dll, and also includes the project-used ModuleAPI_C.lib, along with third-party support libraries ACE.dll and PCOMM.DLL.

The 64-bit dynamic library is also named ModuleAPI_C.dll, includes the project-used ModuleAPI_C.lib, and requires only the third-party support library PCOMM.DLL. The ModuleReader.h header file is the sole header used by C language development toolkits across all platforms. This document primarily introduces the common structures, enumerations, and interface functions within this header file. All code snippets in the document assume that hreader is a reader instance handle created by the InitReader_Notype function. It also assumes that err is defined as the READER_ERR enumeration type returned by interface functions.

Note: Interface functions with the suffix "_BaseType" indicate that the parameters use basic types, facilitating calls from non-C languages.
