Inventory

Inventory or inventorying operations generally refer to the process of reading the EPC IDs of tags by polling the antennas according to specified rules. The module's antenna polling sequence starts from the smallest antenna number and switches to the next antenna either when no new tags are detected within a specified duration or when the maximum dwell time for the current antenna is reached. The time threshold for determining a new tag is fixed internally by the reader and cannot be configured, whereas the maximum antenna dwell time can be set using the parameter MTR_PARAM_RF_HOPANTTIME.

Reader inventory methods can be broadly categorized into two types. One is referred to as "Synchronous/Normal/Buffered" mode inventory, which is a blocking, synchronous approach. The other is Fast mode inventory, a non-blocking method. Fast mode can be further subdivided based on the reader's capabilities. Readers based on the R2000 chip and the E-series chip support Fast mode inventory, with the E-series also supporting an enhanced variant known as Ex Fast mode inventory. Both modes require active calls to specific API interface functions.

The API also supports an asynchronous inventory method, which is encapsulated using an internal thread. Tag data and other information are returned via callback functions.

During inventory operations, in addition to acquiring the tag's EPC ID, embedded read commands can be configured. This allows reading data from a specified memory bank by setting the parameter MTR_PARAM_TAG_EMBEDEDDATA. When a read command is embedded in the inventory operation, the returned tag data will include the additional read data. The EmbededDataLen member of the TAGINFO structure indicates the length (in bytes) of this additional data. If no read command is embedded or if the embedded read command fails, this member will be 0, indicating no additional data is present. This setting only takes effect for operations on Gen2 tags.

Inventoried tags are stored in a TAGINFO object. The EpcId member represents the tag's EPC ID as a byte array. To display the EPC ID as a hexadecimal string, the function Hex2Str should be used for conversion. Other commonly used conversion functions include Str2Hex and Str2Hex.
