Tag operation

The memory of a Gen2 tag is divided into four banks, namely Bank 0, Bank 1, Bank 2, and Bank 3.​
​Bank 0​​, also referred to as the Reserved Memory Bank, stores the Access Password and the Kill Password. Each password is 32 bits in length.​
Bank 1​​, known as the EPC Bank, contains a CRC field (16 bits), a PC field (16 bits), and the EPC code (which can be up to 496 bits long, though 96 bits is the most common length). The EPC code, or Electronic Product Code, is used to identify the item associated with the tag.​
Bank 2​​, called the TID Bank, stores a globally unique serial number that identifies the tag itself. Common lengths are 64 bits or 96 bits.​
​​Bank 3​​, the User Memory Bank, has a variable capacity that differs across tag manufacturers. Many tags may not even include a Bank 3.

The memory in different banks is accessed by tag operation functions in units of blocks, with each block comprising 16 bits. Addressing starts from block 0. For example, the Kill Password occupies blocks 0 and 1 in Bank 0.

All tag operation functions can be assigned a timeout period. If the operation completes within the specified timeout, the function returns early. Otherwise, the function will block for at most the duration of the timeout.
For inventory operations, multiple antennas can be used to inventory one or more tags. However, basic tag operations—such as read, write, lock, and kill—can only be performed using a single antenna on a single tag. If multiple tags are present in the antenna’s field, the first tag that responds to the reader will be the one operated on. Therefore, to ensure that an operation is applied to a specific tag, there should be only one tag within the antenna’s field, or filtering conditions should be configured.

Note: Do not attempt tag operations using a port that is not connected to an antenna, as this may cause hardware damage to the reader.
