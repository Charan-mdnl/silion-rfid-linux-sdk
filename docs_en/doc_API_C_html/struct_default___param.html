◆ customkey

void* Default_Param::customkey

When key=MTR_PARAM_SAVEINMODULE,then customkey can there be specific values.Currently supported (also requiring module support)：

  • "modulesave/hpupload" customize Hp function and proactively upload
  • "modulesave/antport" default antenna port for power on
  • "modulesave/default" restore the initial power on default value

◆ isdefault

int Default_Param::isdefault

Whether configured as default,when modifying custom default configurations, it should be set to false, true will be set as default, and the val value will be ignored at this time

◆ key

Mtr_Param Default_Param::key

Enum type for predefined power-up default parameter keywords,Currently supported enum type as：

  • MTR_PARAM_SAVEINMODULE_BAUD ,val type is int *, and the value can be 4800,9600,19200,38400,57600,115200,230400,460800,921600
  • MTR_PARAM_FREQUENCY_REGION ,val type is unsigned char*
  • MTR_PARAM_RF_ANTPOWER ,val type is unsigned short *,Sequence: antenna number1, read power1, write power1, antenna switching time1 (now fixed at 0), antenna number 2, read power 2...
  • MTR_PARAM_POTL_GEN2_SESSION ,val type is int*
  • MTR_PARAM_POTL_GEN2_Q ,val type is int*
  • MTR_PARAM_POTL_GEN2_TARGET ,val type is int*
  • MTR_PARAM_SAVEINMODULE ,val type is int* if key is MTR_PARAM_SAVEINMODULE,then it is the extended key, customkey定define keyword names.
