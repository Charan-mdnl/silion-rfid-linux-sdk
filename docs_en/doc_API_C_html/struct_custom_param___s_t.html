The byte length of pCusParams, note that when pCusParams is the input parameter, CParamlen is only the length of the specific parameter value after 50 bytes; When pCusParam is an out parameter, CParamalen contains the length of the first 50 bytes of the parameter name

◆ pCusParam

void* CustomParam_ST::pCusParam

Contains custom extended parameter keywords and parameter values. The first byte is the keyword in the following three situations,It determines the subsequent bytes :

  • The first byte is 0x01 Perform carrier testing.The byte sequence of parameter values is：enable or disable(one byte,1 means enable,0 means disable)+antenna id(one byte)+power(2 bytes)+frequency(4bytes).This parameter is only set.
  • The first byte is 0x02 means switch the uhf module to the BootLoader layer.no paramters.This parameter is only set.
  • The first byte is 0x03 means read-write registers,The byte sequence of parameter values is：4 bytes address+4 bytes value.This parameter can be set or get.
    Except the above situations,The first 50 bytes are used for the parameter name,after 50 bytes are for the specific parameter values,the following keywords are now supported：
  • Paramter named"reader/rdrdetails",means to get hardware information of the reader and returning a byte sequence of parameter values:
    16 bytes reserved+4 bytes hardware version+12 bytes serial number. so CParamlen of pCusParam is not less than 50+32 bytes.
  • Paramter named "reader/moduleinfo",means to get detailed information about the reader and return parameter values in byte order:
    16 bytes reserved+16 bytes version infomation(4 bytes Bootloader+4 bytes hardware version+4 bytes firmware date+4 bytes firmware version)+12 bytes serial number. so CParamlen of pCusParam is not less than 50+44 bytes.
  • Paramter named "reader/macaddr", means to get the MAC address and returning 6 bytes parameter value.
  • Paramter named "reader/infacthardwareinfo",means to get internal hardware information (module support required).Return parameter value byte sequence: 1 byte indicates whether there is internal information+64 bytes of information.
  • Paramter named "Reader/Ex10fastmode",means to switch between normal fast mode and Ex fast mode. Parameter value 22 bytes, byte sequence is：mode(one byte，1 means Ex-fastmode，0 means fastmode)+1 byte(fix value is 20)+ scene one byte(0 is more tags in cabinet scene,1 is less tags scene)+19 bytes(all is 0 now).This parameter can be set or get.
  • Paramter named "tagcustomcmd/fastid"，means whether to enable fastid function，tag support needly.parameter value byte sequence：enable or disable(one byte，1 means enable，0 means disable).This parameter can be set or get.
  • Paramter named "tagcustomcmd/tagfocus"，means whether to enable tagfocus function. parameter value byte sequence：enable or disable(one byte，1 means enable，0 means disable).This parameter can be set or get.
  • Paramter named "gen2op/multiembeddeddata"，mean to set multiple additional data items,the total number of blocks cannot exceed 56.parameter value byte sequence:additional items count(1 byte)+password(4 bytes)+additional1 bank(1 byte)+ additional1 address(4 bytes)+additional1 block count(1 byte)+additional2 bank(1 byte)+additional2 address(4 bytes)+additional2 block count(1 byte)+additional3 block count(1 byte)+....
  • Paramter named "gen2op/settag2selected"，reserve
  • Paramter named "blf/asyncinvparams"，reserve
  • Paramter named "rfcommon/anthoptab"，reserve
  • Paramter named "rfcommon/hopmode"，reserve
  • Paramter named "Reader/Savestandby"，Indicates whether to switch the reader to the Bootloader layer when closing it. the parameter value is one byte，bytes sequence：enable or disable(one byte，1 means enable，0 means disable).This parameter can be set or get.
  • Paramter named "R2000/oemregister"，Read and write OEM registers， the parameter value is 4 bytes，bytes sequence：4 bytes address+4 bytes value.This parameter can be set or get.
  • Paramter named "gen2op/cc_33_cmd"，reserve
  • Paramter named "Reader/impinjfastmode"，reserve
  • Paramter named "Reader/dutycycle"，mean to set duty cycle parameter, the parameter value is 4 bytes. bytes sequence:2 bytes the time of inventory before config duty cycle ,in milliseconds+:2 bytes the benchmark time for duty cycle execution,in milliseconds.
  • Paramter named "Reader/powermode",set to reader power mode,parameter value is 1 byte：1 byte mode;0x00 FULL Power， 0001 MIN_SAVING, 0x02 MED_SAVING, 0x03 MAX_SAVING.
  • Paramter named "Reader/rssifilter"， reader internal RSSI filter config， the parameter value is 4 bytes，bytes sequence：when get iinternal filter，4 bytes are all 0;when clear internal filter，one byte 0x01(set)+3 bytes are all 00;
    when set the module internal filter ：one byte 0x01+1 byte 0xAA+1 byte signed RSSI value+1 byte 0x00.
