ModuleAPI_C简介

ModuleAPI C 为C语言开发接口。可以应用于windows，linux，android等操作系统平台。

ModuleAPI C以动态库的形式对接，供开发者引用。开发者可以使用动态库中的接口函数驱动读写器工作。 不同的平台ModuleAPI C动态库名称略有不同，linux操作系统平台为libModuleAPI.so,分别提供基于x86-64指令架构的CPU编译的32位或64位的动态库。 Linux系统基于其它指令架构的动态库则需要联系技术人员。window操作平台则提供基于x86-64的32位或64位的动态库， 32位动态库为ModuleAPI_C.dll,也提供工程使用的ModuleAPI_C.lib,同时需要第三方支持库ACE.dll, PCOMM.DLL。 64位动态库为ModuleAPI_C.dll,也提供工程使用的ModuleAPI_C.lib,同时仅需要第三方支持库PCOMM.DLL。

ModuleReader.h 头文件是所有平台C语言开发工具包使用的唯一头文件。本文档主要介绍该头文件中常见结构体和枚举类型，以及接口函数。 文档中的所有示例代码片段都假定hreader是InitReader_Notype函数创建的读写器实例句柄。假定err定义为接口函数返回的枚举类型 READER_ERR。

注意：带_BaseType的后缀接口函数，表示参数使用基本类型，利于非C语言调用。
