标签操作

Gen2标签的内存被分成四个bank，分别是bank0，bank1，bank2，bank3，bank0又称为保留区，存放着访问密码和销毁密码，每个密码都有32bits。 bank1又称为EPC区，其中又含有CRC字段（16bits），PC字段（16bits），EPC码（最长可到达496bits，一般为96bits）。EPC码称为物品编码，用于标识标签关联的物品。 bank2又称为TID区，用于标识自身为全球唯一的序列号，常见64bits/96bits。bank3又称为USER区，容量是不定的，不同品牌的标签容量不同，也有很多标签没有bank3。

Gen2标签内存结构图

    
    
序号 BANK 描述(1块Block=2字节Byte=16位bit)(块地址从0开始)
0 保留区 第0~1块Kill销毁密码。第2~3块Access访问密码。
1 EPC 第0块TagCRC_EPCCRC，第1块PC，从第2块开始后续为EPCID。
2 TID 块地址从0开始，首块即第0块。
3 USER 块地址从0开始，首块即第0块。
    
    

不同的bank中，标签操作函数操作的最小内存单元为块，每个块是16个bit，从第0块开始编址，例如对于销毁密码而言，它所占有的内存区域为bank0的第0块和第1块。

标签操作函数都可以指定一个超时时间，如果操作在超时时间内完成，函数就会提前于超时时间返回，否则函数最长会阻塞直到超时。
对于盘存标签操作可以指定多个天线盘存一个或多个标签；对于标签基本操作：读，写，锁，销毁只能指定一个天线操作一个标签。如果天线场中有多个标签， 第一个应答读写器的标签将会是被操作的标签。因此如果想要准确将一个操作作用到指定的标签中，应保证在天线的场区中只有一个标，或者通过设置过滤条件。

注意：不要选择没有接天线的端口进行标签操作，这样可能导致读写器的硬件损坏。
