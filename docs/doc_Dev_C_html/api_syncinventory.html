同步普通盘存

使用TagInventory_Raw和GetNextTag 函数实现同步盘存。TagInventory_Raw函数需要传入一个指定的时间来盘存标签并缓冲在读写器内部，调用后将阻塞直到超时时间结束，结束后返回读取标签个数。原则上阻塞期间内不允许调用其它RFID操作接口。 使用GetNextTag函数从缓冲区取出。返回多少个标签就要调用多少次GetNextTag。

当用户需要执行短时间的盘存操作，并且标签数量不是很多时，同步盘存是一个不错的选择。

由于读写器内部缓冲标签个数有限并且TagInventory_Raw为阻塞函数，因此使用普通模式盘点较多标签时候，一般结合定时器或者线程来循环调用，考虑散热和硬件损耗则可以加上适当休闲时间。

注意：除了以前m6e类型读写器在调用上述函数之前必须先设置MTR_PARAM_TAG_INVPOTL 参数，否则会返回错误。现读写器此项有默认值，为非必须配置项。
