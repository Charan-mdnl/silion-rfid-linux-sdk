盘存

盘存或者盘点操作一般是指按规定天线轮询读取标签的EPC ID。 现模块的天线轮询规则是从天线号值最小开始，指定时间内无新增标签或者达到天线最大的驻留时间而切换下一个天线。 新增标签判定时间由读写器内部指定无法设置，而天线最大驻留时间可以配置MTR_PARAM_RF_HOPANTTIME。

读写器盘存方式大致分为两类，一类是称为“同步/普通/缓冲” 模式盘存，这是一种阻塞的同步方式。 另一类是快速模式盘存，非阻塞的方式，快速模式根据读写器性质不同还可以细分。基于R2000芯片，E系列芯片的读写器支持快速模式盘存， E系列还支持E系列快速模式盘存称为Ex快速模式。这两种方式接口都是需要主动调用API接口函数。

API还支持异步盘存方式，由内部线程进行封装。标签以及其它数据通过回调方式返回。

在执行盘存操作的时候，除了可以取得标签的EPC ID，还可以嵌入附加的读指令，允许通过设置MTR_PARAM_TAG_EMBEDEDDATA来读指定bank中数据；。当在盘存操作中嵌入了读指令后，返回的标签数据就带有了附加数据， TAGINFO的成员 EmbededDatalen将指示附加数据的字节数，在没有嵌入读指令或者嵌入的读指令执行失败时，此成员都为0，表示没有附加的数据。此设置只针对gen2标签操作起作用。

盘存到标签将存储于TAGINFO对象中，其中EpcId 为标签EPCID是字节数组类型， 一般以16进制字符串方式显示EPCID，所以需要使用函数Hex2Str来转换。
常用转换函数还有Str2Hex和 Str2Binary。
