◆ Lock_Type

enum Lock_Type

Gen2协议标签的锁类型

对区锁定后，根据不同区，不同锁有不同的结果。解锁与锁定是一对逆操作。 锁定可以认为对Gen2存储区的访问限制或者访问保护。解锁即解除区的密码锁定操作。解锁是锁定的逆操作，但永久锁定操作不可逆。

  • 锁定保留区后，不可读写。不解锁情况下，读写需要密码。
  • 锁定EPC，USER区可读不可写。不解锁情况下，写需要密码。
  • TID区一般出厂已永久锁定，永久锁定的TID则只能读即使带密码也不能写，而且不能解锁。
  • 若进行了永久锁定，密码也无效。保留区不可读写，其它区不可写。
  • 若进行了永久不锁定。则无法再锁定标签存储区。

READER_ERR EXTERN AsyncStartReading ( int hReader,
int * ants,
int antcnt,
int option )

开始快速模式盘存

开始快速模式盘存后，读写器一直处于盘存状态。非异常或特殊情况不停止。

参数
[in]hReader已初始化的读写器句柄
[in]ants操作天线 从1开始
[in]antcnt操作天线数组个数
[in]option数据类型为int，从高位起， 第一字节，BIT0控制是否启用天线组轮询一遍通知功能(使能多个天线，则每轮询完一遍天线，就会上传一个通知帧的数据包)。1启用 0不启用。启用对应值 0x01。 启用了轮询通知功能，那么会在轮询一遍天线后产生一个虚拟标签，其epclen长度为1。
第二，第三字节为metaflag，即数据返回项标记。
第四字节低四位为停顿百分比，0对应0，1对应5%，2对应10%.....10对应50%，11:55，12:60，13:70%,14:80%,15:90。
第四字节高四位中，BIT7控制是否启用每隔15秒向主机发送心跳包功能，1启用 0不启用。启用对应值 0x80。
BIT6控制是否启用无新增标签自动停止盘存功能（当使用了单天线，且天线驻留时间到或无法读取到新的标签，则停止盘存）。1启用 0不启用。启用对应值 0x40。
返回
操作结果
//返回所有数据项
int metaflag = 0;
metaflag |= 0X0001;//返回读次数
metaflag |= 0X0002; //返回RSSI信号
metaflag |= 0X0004; //返回天线号
metaflag |= 0X0008;//返回频点
metaflag |= 0X0010;//返回时间戳
metaflag |= 0X0020;//返回相位值
metaflag |= 0X0040;//返回标签协议值
metaflag |= 0X0080;//返回附加数据
//停顿比：50%(读时间和间隔比例)
int option = (metaflag << 8)| 10;
//启用心跳
option=option|0x80;
//启用自动停止功能
